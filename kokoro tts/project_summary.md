# 프로젝트 리서치 및 기술 스택 요약 (2026-02-16)

## 1. 프로젝트 개요
* **목표:** '이터널 리턴' 게임 관련 디스코드 봇 (**debi-marlene**) 개발 및 상업적 배포(수익화)
* **핵심 기능:** 실시간 데이터 조회, AI 기반 인터랙션, 고품질 TTS 기능 탑재
* **개발자 프로필:** CV, VLM, TTS 등 AI 모델 활용 및 서비스화에 집중

## 2. 개발 환경 (Terminal & CLI)
### 터미널 비교 분석
| 도구 | 상태 | 핵심 특징 및 이슈 |
| :--- | :--- | :--- |
| **iTerm2** | 기존 사용 | Claude Code 4분할 시 스크롤 요요(Yo-yo) 현상 및 렌더링 지연 발생 |
| **Ghostty** | 테스트 중 | GPU 가속(Metal) 지원, **Synchronized Output**으로 Claude Code 스크롤 문제 해결 최적화 |
| **Warp** | 검토 중 | **Blocks** 기반 UI, 자체 AI 에이전트 내장. 단, TUI 앱(Claude Code)과 레이아웃 충돌 가능성 있음 |

* **특이사항:** Claude Code 사용 시 터미널 레이아웃이 뭉개지는 현상 대응 필요.

## 3. TTS 모델 조사 (상업용 및 한국어 타겟)
### 비자기회귀(Non-Autoregressive, NAR) 모델 중심
* **이점:** 병렬 생성 방식으로 추론 속도 $O(1)$ 달성, 서버 비용 절감 및 실시간성 확보.

| 모델 | 라이선스 | 특징 |
| :--- | :--- | :--- |
| **Kokoro-82M** | **Apache 2.0** | **초소형(82M)**, 극도의 속도, 한국어(v1.0+) 지원, ONNX 변환 가능 |
| **MeloTTS** | **MIT** | 안정적인 다국어 지원, VITS 기반 고속 추론, 설정 및 배포 용이 |
| **MaskGCT** | 비상업용 | 제로샷 클로닝 성능은 우수하나 **수익화 프로젝트 적용 불가** |

* **최종 검토:** 상업적 이용이 자유로운 **Kokoro-82M**을 최우선 적용 후보로 선정.

## 4. 최신 기술 동향 및 도구
* **Chrome DevTools MCP:** AI 에이전트가 브라우저 디버깅 및 콘솔 로그를 직접 분석하는 환경.
* **WebMCP:** 픽셀 기반이 아닌 API 기반의 차세대 브라우저 제어 표준 모니터링 중.

## 5. 향후 과제 (에이전트 요청용)
1. **Kokoro-onnx** 기반의 한국어 고효율 추론 파이썬 스크립트 작성.
2. 디스코드 봇 배포를 위한 경량화된 **Docker** 구성안 도출.
3. 터미널 내 Claude Code 레이아웃 최적화 설정값(Window step resize 등) 검증.

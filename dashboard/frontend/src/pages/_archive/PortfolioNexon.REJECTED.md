# PortfolioNexon — 아카이브

넥슨 [플랫폼본부] 컴퍼니솔루션실 웹 애플리케이션 엔지니어(계약직) 지원 탈락 (2026-04, 사유 비공개).
라이브 라우트(`/portfolio/nexon`)에서 제거하고 참고용으로만 보존.

## 탈락 원인 분석 (유력 순)

### 1. 서류 필터링 단계 (가장 큼)
- **학력·경력 조합**: 신구대 시각디자인과 재학중(당시) + 비전공 + 경력 없음(6개월 사이드 프로젝트 1건).
  대기업 ATS/HR 1차 필터에서 사람이 내용을 보기 전에 자동 컷 가능성 높음.
- 포트폴리오 완성도와 무관하게 서류 단계에서 끊겼을 공산이 크다.

### 2. 도메인 미스매치
- 컴퍼니솔루션실 = **사내 임직원용** 엔터프라이즈 웹(전자결재/그룹웨어/ERP 성격).
- 포트폴리오는 **B2C 게이머 커뮤니티 Discord 봇**.
- "동일 아키텍처를 사내 솔루션으로 이식 가능"(PortfolioNexon.tsx:640) 주장했지만 엔터프라이즈 핵심 요구가 프로젝트에 전무:
  - SSO/SAML/OIDC, AD 연동, RBAC
  - 감사 로그, 개인정보보호법 준수, 접근 통제
  - 레거시 시스템 통합, 대규모 사내 트래픽/SLA
- Discord OAuth2 경험은 사내 SSO 복잡도 증빙이 못 됨.

### 3. 기술 스택 미스매치
- 포폴 스택: Flask + React + GCP + Modal + LangGraph → 스타트업/사이드 스택.
- 넥슨 사내 솔루션은 통상 Java/Spring/Oracle/사설 K8s 중심. GCP·Modal은 "가점"이 아니라 "우리 스택 아님"으로 읽힘.
- "Makefile CI/CD"(PortfolioNexon.tsx:690) 라고 썼지만 본문에 **CI 없음 + CD만**이라고 스스로 명시 → CI/CD 실경험 부재 시그널.

### 4. Claude Code 과노출이 역효과
- Hero 통계 4개 중 하나 = "AI-First · Built with Claude Code" (PortfolioNexon.tsx:1022)
- resume.md 소개 첫 문장 = "Claude Code를 활용해 ..."
- resume.md 하단 = AI 도구 활용 타임라인 섹션
- 재학생/신입이 AI 도구를 전면에 세우면 "본인 역량 vs 도구가 짠 코드" 구분이 안 됨.
  엔터프라이즈 유지보수 관점에서 리스크 신호로 읽힘.

### 5. 자기 깎아내리기 명문화
- PortfolioNexon.tsx:696 우대사항 카드 — "1인 개발이라 팀 협업 경험은 제한적 — 입사 후 본격화 예정."
- 솔직한 건 좋지만 포폴 본문에 이 문장이 있으면 **팀 협업이 기본인 포지션에서 자진 탈락 선언**.
- 차기 버전에서는 삭제하거나 "오픈소스 PR·코드리뷰" 등 우회 증빙으로 교체.

### 6. 정량 임팩트 부재
- JD 매칭 근거가 구현 사실 나열 위주.
- "수백 ms 단축"(PortfolioNexon.tsx:664) 같은 정성 표현만 있고, **Before/After 절대값·MAU/DAU·가동률·P95·월 비용** 같은 엔터프라이즈 신뢰 지표가 없음.

### 7. 포맷 자체의 부담
- GSAP pin 가로 스크롤 + Lenis + Framer + Mermaid + 라이트/다크 + JD variant 탭.
- 심사자는 짧은 시간에 여러 명을 본다 — 로딩 느리고 스크롤 강제되면 끝까지 안 봄.
- 사내 솔루션 팀은 꾸밈보다 가독성·정확성을 우선시.
- 추가로 **company/ax 듀얼 탭** = "재활용 지원 티남" 시그널로도 해석될 수 있음.
